# ECHO QA 检查清单

> **为什么维护这份清单？**
> 
> ECHO 是一个零依赖、无构建工具的浏览器扩展，目前尚未引入自动化测试。功能模块之间存在大量隐式耦合——特别是标签管理模块，它通过事件监听链（`onCreated → onActivated → onMoved → onRemoved`）维护全局状态，任何一处改动都可能在看似无关的场景中引发回归。
> 
> 考虑到这一点，开发者决定维护一份结构化的手动测试检查清单，作为每次发版前的质量关卡。虽然不能替代自动化测试，但至少确保核心路径在每个版本都被系统性验证过。
> 
> 清单基于对全部源码的审计和已知薄弱点整理，按优先级分三层。

---

## 第一层：冒烟测试（2 分钟）

每次改完代码、重新加载扩展后，先跑这 4 项。如果任何一项失败，说明核心通信链路断了，不要继续测其他的，先定位问题。

| # | 操作 | 预期结果 | 验证的链路 |
|---|------|----------|------------|
| S1 | 打开新标签页 | NTP 正常加载，不白屏，壁纸或 Low Poly 背景显示 | ntp.js 加载正常 |
| S2 | 点击扩展图标 | 设置页打开 | background.js → options.html |
| S3 | 任意网页上 Ctrl+滚轮 | 页面中央显示缩放百分比指示器 | content.js → background.js（getZoom/setZoom）→ content.js |
| S4 | 任意网页上右键+滚轮 | 切换到相邻标签，松开右键不弹菜单 | content.js → background.js（switchTab）→ tabs API |

## 第二层：功能回归测试（10-15 分钟）

按模块逐个验证。标注 ⚠️ 的是审计中发现的高风险项——改动 background.js 后务必重点测。

### 标签管理 ⚠️

这是最复杂的模块，涉及 Promise 队列、per-window 状态缓存、6 场景关闭激活矩阵。

| # | 操作 | 预期结果 | 为什么重要 |
|---|------|----------|------------|
| T1 | 在网页上 Ctrl+Click 打开一个链接 | 新标签出现在当前标签**右侧**（不是最右） | 基本的新标签位置控制 |
| T2 | 快速连续 Ctrl+Click 打开 3 个链接 | 3 个新标签按点击顺序排列在当前标签右侧，不乱序 | Promise 队列串行化 |
| T3 | 关闭一个**中间位置**的标签 | 焦点跳到左侧标签 | 关闭激活策略：场景 6 |
| T4 | 关闭**最左边**的标签 | 焦点跳到右侧（因为左边没有了） | 关闭激活策略：场景 5 |
| T5 | 关闭**最右边**的标签 | 焦点跳到左侧（浏览器默认行为，不应干预） | 关闭激活策略：场景 4 |
| T6 | 关闭一个**非活跃**标签 | 当前焦点不变 | 关闭激活策略：场景 2 |
| T7 | 拖拽移动一个标签到新位置 → 再打开新链接 | 新标签出现在移动后的当前标签右侧 | onMoved 同步 baseTabIndex |
| T8 | 打开第二个窗口 → 在窗口 B 里操作标签 | 窗口 A 的标签不受影响 | per-window 状态隔离 |

## 第三层：边界场景与已知陷阱

这些是审计中发现的特定边缘情况。不需要每次都测，但在涉及相关模块的改动后应针对性验证。

| # | 场景 | 预期结果 | 触发条件 |
|---|------|----------|----------|
| E1 | 页面缩放到 150% 后按 Ctrl+B | 悬浮搜索框视觉大小与 100% 时一致 | 修改 search-box.js 后 |
| E2 | 页面缩放到 150% 后右下角关联搜索出现 | 推荐气泡视觉大小与 100% 时一致 | 修改 related-search.js 后 |
| E3 | NTP 上 Ctrl+滚轮缩放 → 关闭 NTP → 重新打开 | 缩放比例保持（CSS transform 持久化） | 修改 ntp.js 缩放逻辑后 |
| E4 | 打开 edge://settings 等浏览器内置页面 → F2/F3 | 内置页面被**跳过**，切到下一个可切换的标签 | 修改 isSwitchableTab 逻辑后 |
| E5 | 连续快速关闭多个标签 | 每次都激活正确的左侧标签，不乱跳 | ⚠️ isProcessingRemoval 标志竞态 |
| E6 | 壁纸收藏超过 50 张 | 收藏正常保存和加载 | chrome.storage.sync 配额（8KB/key） |
| E7 | 完全断网状态下打开 NTP | 壁纸仍然显示（来自 IndexedDB 缓存） | 7 天内打开过 NTP 的前提下 |
| E8 | 设置页修改标签管理选项 → 立即测试 | 新设置立即生效，无需重启扩展 | chrome.storage.onChanged 实时同步 |
| E9 | Alt+点击有防盗链的图片（如微博图片） | 仍然保存成功 | ⚠️ declarativeNetRequest 临时 Referer 注入 |

## 使用说明

- **发版前**：至少完整跑一遍第一层 + 第二层
- **小改动后**：跑第一层 + 相关模块的第二层项
- **改动了 background.js**：⚠️ 标签管理全部测一遍（T1-T8），加上 E5
- **改动了消息处理逻辑**：冒烟测试全部 + S3/S4 + B1/B2 + R1
