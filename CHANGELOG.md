# 更新日志 / Changelog

## 1.0.7 - 2026-03-19
- 更新扩展图标：基于新设计的高分辨率源图重新生成 16/48/128 三种尺寸图标（Lanczos 重采样 + 锐化），替代旧图标

## 1.0.6 - 2026-03-10
- 新增自定义壁纸上传：在壁纸库收藏面板中点击「+」卡片即可上传本地图片作为壁纸，上传后自动加入收藏、锁定并展示；支持 JPG/PNG/WebP 格式，最大 20MB，上限 10 张
- 自定义壁纸在收藏网格中常驻显示「本地上传」标签以区分 Bing 壁纸，信息卡片（标题/版权）自动隐藏
- 自定义壁纸仅存储在本地 IndexedDB，不参与备份导出/导入，跨设备同步时静默跳过
- 优化自定义壁纸加载速度：上传时自动压缩为 JPEG（最大 3840×2160），使 IndexedDB 存储体积与 Bing 壁纸一致，消除刷新新标签页时本地壁纸明显慢于在线壁纸的问题；已有壁纸首次显示后自动后台迁移
- 优化壁纸缓存清理：延迟执行并改用游标逐条遍历，避免 readwrite 事务阻塞壁纸显示
- 修复关闭热搜榜后打开新标签页时品牌区和搜索框明显向下跳动的问题：通过同步脚本在首帧前读取 localStorage 预设布局位置，消除异步加载导致的闪烁
- 更新 Bing 壁纸历史数据至 2026-03-10（新增 23 张壁纸，含春节、元宵节等中国特色壁纸）
- README 新增信任声明：完全开源，完全免费，无广告、无弹窗、无打扰，不碰任何隐私
- 修复隐私保护指引中 LICENSE 链接在扩展内打开会直接下载文件的问题，改为指向 GitHub 在线页面
- 修正 README 第三方服务声明：从"均非 Microsoft/Edge 官方服务"改为"均通过公开 API 访问，未使用任何内部接口"，避免对 Bing 壁纸等微软自有服务的描述不准确

## 1.0.4 - 2026-03-04
- 修复 Alt+点击快速保存图片在 Bing 图片搜索详情页（大图预览模式）无效的 bug：Bing 使用全屏 iframe（insightsOverlay）覆盖在图片上方，导致父窗口的事件监听器完全收不到点击事件。修复方案：manifest 启用 `all_frames: true` 使 content.js 注入 iframe，iframe 内仅运行 Alt+click 保存逻辑，跳过手势/拖拽等其他功能；同时增加 `elementsFromPoint` 穿透透明 DIV 覆盖层查找图片的能力
- 修复新标签页热搜榜在中等宽度屏幕（≤1400px）下内容溢出背景的布局 bug：CSS grid 和 JS 渲染均从固定 4 列改为响应式 3/4 列，并在窗口大小变化时自动重新渲染
- 优化按钮文案避免歧义：「收藏」→「收藏壁纸」/「已收藏壁纸」，「设为壁纸」→「锁定壁纸」/「已锁定壁纸」，同步更新按钮固定宽度和所有相关 Toast 提示
- 移除新标签页品牌区域的英文副标题（Edge Chinese Helper & Optimizer），简化视觉呈现
- 新增热搜榜单数据量过滤：API 返回不足 20 条数据的榜单自动隐藏且不显示翻页圆点，箭头和滚轮切换时自动跳过
- 优化悬浮搜索框（Ctrl+B）热搜交互：移除上下相邻行溢出显示，改为 hover 时容器外侧显示小箭头提示；修复 hover 后文字蓝色不恢复的 bug；hover 响应区域扩大到整个热搜容器
- 悬浮搜索框热搜推荐默认关闭（原为默认开启）
- 统一简化 NTP、设置页、FRE 四步页面的品牌区域：移除英文副标题，调整 logo 与文字尺寸比例
- 简化 manifest 插件描述文案
- 设置页实验室从「个性化」中独立为页面底部单独区块，调整卡片结构与其他设置项一致
